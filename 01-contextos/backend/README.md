# Contexto Backend

Esta carpeta mantiene el contexto del backend. El backend aún no está definido, por lo que cualquier selección tecnológica debe registrarse como ADR antes de asumirse como vigente.

## Convención

- Archivo: `YYYY-MM-DD-slug-descriptivo.md`
- Template: `99-plantillas/contexto.md`
- Tags habituales: `backend`, `api`, `database`, `arquitectura`, `seguridad`

## Qué Documentar Aquí

- Opciones de backend evaluadas.
- Contratos API.
- Estrategia de persistencia.
- Autenticación y autorización.
- Integraciones servidor-servidor.
- Operación, deploy y observabilidad.

## Regla Importante

No asumir framework, lenguaje, base de datos ni plataforma backend sin una decisión ADR aceptada.
