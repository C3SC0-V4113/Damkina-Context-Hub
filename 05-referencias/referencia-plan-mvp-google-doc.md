## Ideas Relevantes Para Damkina

- El MVP debe validar la conversión de una ubicación real en recomendaciones agrícolas útiles.
- El frontend se plantea en Flutter.
- El documento menciona Node.js como propuesta técnica para backend, pendiente de decisión ADR.
- La ubicación debe tratarse como perfil agroambiental, no solo como pin de mapa.
- El motor de recomendación debe explicar por qué un cultivo fue recomendado.
- La clasificación Köppen puede requerir dataset externo, fuente geoespacial o tabla precalculada.
