# Catálogo De Tags

Tags iniciales permitidos para clasificar contexto.

| Tag | Uso |
| --- | --- |
| `frontend` | Contexto general de la capa cliente. |
| `flutter` | Decisiones o contexto específico de Flutter. |
| `backend` | Contexto o decisiones del backend. |
| `api` | Contratos, endpoints, protocolos o comunicación entre sistemas. |
| `auth` | Autenticación, autorización, sesiones, identidad y permisos. |
| `database` | Persistencia, modelos de datos, consultas, migraciones o almacenamiento. |
| `arquitectura` | Decisiones estructurales, patrones, límites y componentes principales. |
| `dominio` | Conceptos, entidades, reglas y lenguaje del negocio. |
| `producto` | Alcance funcional, objetivos, usuarios y necesidades de producto. |
| `ux` | Experiencia de usuario, navegación, interacción y diseño funcional. |
| `integracion` | Servicios externos, APIs de terceros y sincronización entre sistemas. |
| `operaciones` | Deploy, observabilidad, ambientes, soporte y mantenimiento. |
| `seguridad` | Controles de seguridad, riesgos, protección de datos y cumplimiento. |
| `performance` | Rendimiento, escalabilidad, latencia, memoria y optimización. |
| `ia-contexto` | Instrucciones, memoria y estructura pensada para agentes de IA. |

## Reglas Para Tags Nuevos

- Usar minúsculas.
- Usar guiones medios si son varias palabras.
- Evitar sinónimos si ya existe un tag equivalente.
- Agregar el nuevo tag a esta tabla antes de usarlo ampliamente.
